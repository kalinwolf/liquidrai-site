Core

What is LiquidRai?
A real asset project in Chiang Rai combining land/build-out + goat operations, with an optional tokenized participation layer for reporting/distribution integrity.
What’s the main objective?
Predictable cash-flow and repeatable execution. Not hype, not “community vibes” — proof and operations.
How do I evaluate it?
Start with Project + Timeline. Then request the pack for numbers, assumptions, and disclosures.

Token

Are tokens required?
No. The operation stands on its own. The token layer is optional and must be compliant.
Do tokens create yield?
No. Operations create yield. Tokens only track participation and help enforce reporting/distribution rules.

Operations

What are the main risks?
Execution delays, input cost volatility, animal health events, regulatory changes, and market pricing shifts. We mitigate via gates + KPI tracking + staged spend.
